# 战略校验 Skill：从 BSC 到 OKR 的红队审计 (2026 Pro版)

> **核心哲学（红队审计）**：**战略的默认结果是扭曲与失败**。
> 本 Skill 不负责鼓舞士气，只负责**“事前验尸” (Pre-mortem)**。我们将以**极度悲观**的视角，假设每一个指标都会被误解、每一个口径都会被注水、每一个考核都会被博弈。
> **针对喜播 2026**：我们正处在“第一曲线解毒”与“第二曲线创业”的生死关口，任何虚假繁荣的指标（如带毒的 GMV）都是致死毒药。

## ⚠️ 审计官人格设定
在执行本 Skill 时，请切换至以下人格：
1.  **亚马逊审计员 (Data Police)**：不相信形容词，只相信定义了分母和分子的 SQL 逻辑。
2.  **阿里政委 (Value Guard)**：敏锐嗅探“野狗”气味——那些为了达成 KPI 而牺牲长期价值观（如“不暴雷”）的行为。
3.  **华为蓝军 (Red Team)**：站在竞争对手或监管机构的角度，攻击我们现有的战略防线。

## 一、 第一层校验：指标物理学 (The Physics of Metrics)

**目标**：确保指标是“真实世界”的投影，而非“数字游戏”的产物。

### 1.1 数据字典审计 (Data Dictionary Audit)
*   **亚马逊原则**：拒绝模糊定义。所有指标必须通过“二阶定义”测试。
*   **审计清单**：
    *   [ ] **定义锁死**：例如“活跃用户”，必须明确是“打开 APP”还是“完成一次核心交付（如练满10分钟）”？
        *   *2026 修正*：对于健康业务，仅“打开”是无效的伪指标，必须是 **“有效依从行为”**。
    *   **时间戳对齐**：指标是 T+0 (实时) 还是 T+1？跨部门对齐了吗？
    *   **排重逻辑**：计算“用户数”时，同一个用户买两门课算 1 个还是 2 个？（必须按 UV 算影响力，按 PV 算工作量，不可混淆）。

### 1.2 计算口径白盒化 (Calculation White-box)
*   **核心逻辑**：任何无法在 3 分钟内解释清楚计算逻辑的指标，都是垃圾指标。
*   **2026 专项审计**：
    *   [ ] **收入 (Revenue)**：
        *   ❌ 错误口径：合同金额 (Contract Value)。这是“狩猎者”视角的虚数。
        *   ✅ 正确口径：**净实收 (Net Cash) - 预估退费准备金 - 渠道成本**。这是 2026 要求的 **“Clean Revenue”**。
    *   [ ] **用户数 (Users)**：
        *   ❌ 错误口径：累计注册用户。
        *   ✅ 正确口径：**本月产生真实健康行为的活跃用户** (第一曲线) / **建立档案且有互动记录的咨询用户** (第二曲线)。

## 二、 第二层校验：反作弊与博弈 (Anti-Gaming & Counter-metrics)

**目标**：预判员工如何“玩弄”指标，并提前堵死漏洞。

### 2.1 哥德哈特定律测试 (Goodhart's Law Test)
> “当一个指标成为目标时，它就不再是一个好指标。”

*   **作弊难度评分 (1-10分)**：给 BSC 中每个指标打分。1分=极易作弊（如自填问卷），10分=极难作弊（如银行流水）。
    *   *红线*：核心战役指标的作弊难度必须 > 8分。
*   **常见作弊手法预演**：
    *   *指标*：客单价。 *作弊*：向无法交付的用户推销高价课（大黑用户）。
    *   *指标*：通话时长。 *作弊*：拨通后挂机或无效寒暄。
    *   *指标*：健康改善率。 *作弊*：只作为筛选器，把难改善的用户踢出分母。

### 2.2 对冲指标机制 (Counter-metrics)
**没有对冲的指标是危险的。** 必须为核心指标设置“刹车”。

| 核心指标 (Accelerator) | **必配的对冲指标 (Bones)** | **2026 战略含义** |
| :--- | :--- | :--- |
| **GMV / 销售额** | **退费率 (Refund Rate) + 有责投诉率** | 防止为了业绩承诺无法交付的结果 (第一曲线解毒) |
| **大黑用户拦截率** | **拦截误伤率 / 申诉通过率** | 防止为了省事把好用户也拦在门外 |
| **人效 (Revenue/Head)** | **NPS (净推荐值) / 员工敬业度** | 防止通过透支品牌和过度压榨来提效 |
| **顾问转化数量** | **顾问留存率 / 顾问平均收入** | 防止“拉人头”式的微商打法，确保顾问能存活 (第二曲线) |

## 三、 第三层校验：战略意图适配度 (Strategic Intent Fit)

**目标**：确保 BSC/OKR 是为 2026 年的“喜播”定制的，而不是通用的模板。

### 3.1 战略一致性扫描 (Based on `xibo2026.md`)
*   **[第一曲线] 解毒性校验**：
    *   有没有指标在**鼓励**“带伤飞行”？（如：只考核总营收，不考核退费）。
    *   是否有**“负向指标”**的明确考核？（如：主动拒绝的“大黑用户”营收额 —— 这是一个伟大的指标，代表我们拒绝了多少诱惑）。
*   **[第二曲线] 启动性校验**：
    *   考核的是“卖课逻辑”（转化率）还是“服务逻辑”（依从性、改变率）？
    *   如果第二曲线满眼都是 GMV，说明战略没落地。**初期核心指标应是：种子用户依从性、顾问活跃度。**

### 3.2 资源锁定校验 (Resource Lock)
*   **2026 关键战役资源**：
    *   “AI 班主任”与“风控系统”的预算是否被**物理隔离**？（不可被其他业务挪用）。
    *   10 万学员清洗与转化的 HC 是否到位？

## 四、 第四层校验：标杆镜像 (The Benchmarks)

**目标**：用巨头的尺子量我们的身高。

### 4.1 华为镜：DSO 与 PBC
*   **DSO (应收账款周转天数) 视角**：
    *   校验我们的现金流健康度。对于教育业务，预收款不是收入，是负债。BSC 里是否有**“现金流/负债比”**的监控？
*   **PBC (个人绩效承诺) 逻辑**：
    *   高管的 OKR 必须签署 PBC，并与**“无退费营收”**挂钩。如果不通过合规考试，PBC 是否自动失效？

### 4.2 阿里镜：价值观双轨制
*   **“野狗”识别器**：
    *   在 BSC/KPI 设计中，是否有一票否决权？
    *   *案例*：业绩第一的 Sales，如果触犯“过度承诺”红线，是否会被系统自动标记并清退？（参考阿里对诚信的零容忍）。

### 4.3 亚马逊镜：Input vs Output
*   **关注可控的输入 (Input)**：
    *   *Output (结果)*：股价、GMV、利润。（这些是滞后的，某种程度上是运气）。
    *   *Input (输入)*：产品丰富度、履约速度、页面加载延迟。（这些是可控的）。
*   **喜播 2026 应用**：
    *   不要只考核“学员健康结果”（Output，受用户体质影响）。
    *   要考核**“顾问有效干预次数”、“健康方案匹配准确率”、“AI 响应速度”**（Input，我们努力就能做到）。

## 五、 第五层校验：BSC 到 OKR 的逻辑穿透 (Alignment)

*(保留原 Skill 核心逻辑，用于确保分解合理)*

### 5.1 纵向因果性 (Vertical Logic)
*   **提问**：如果一线完成了 KPI/OKR，部门的 Key Results 是否**必然**提升？部门 KR 达成，公司的关键战役是否**必然**胜利？
*   **2026 警示**：警惕“虚荣指标”。一线拼命做“销售通话量”（KPI），但全是骚扰电话，导致“品牌信任度”（BSC）下降。

### 5.2 横向协同性 (Horizontal Handshake)
*   **协同握手会**：
    *   产品部（第二曲线）要搞“健康顾问”，市场部（第一曲线）是否背了“输送种子用户”的指标？
    *   如果市场部还只背 GMV，不肯把优质老用户导给新业务，战役必败。

## 六、 终极清单：战略就绪度 (The Final Checklist)

在发布 BSC/OKR 前，必须通过以下 5 个灵魂拷问：

1.  **定义清晰吗？** 能否写出每个指标的 SQL 逻辑？
2.  **数据干净吗？** 所有的 GMV 都是“Clean Revenue”吗？
3.  **对冲了吗？** 每一个增长指标旁边，是否都站着一个风控指标？
4.  **方向对吗？** 指标是在把我们推向“狩猎模式”（如纯销售额）还是“农耕模式”（如依从性）？
5.  **能作弊吗？** 如果你是员工，你能想到 3 种以上不干活也能拿奖金的方法吗？（如果有，重写）。

> **结论**：好的战略校验，不是为了证明战略有多完美，而是为了证明**我们在多大程度上敢于面对现实**。

# 百万元投诉率 · 投诉计算口径（最小颗粒度）

> 为公式 **(外部投诉订单金额/现金流)×60% + (内部投诉订单金额/现金流)×40%** 提供可执行、可审计、防变形的口径定义。战略目标：量化「带风险的收入」，推动净营收与合规双达标。

---

## 一、公式与颗粒度层级

| 层级 | 含义 | 统计对象 |
|------|------|----------|
| **分子** | 投诉订单金额 | 在统计周期内，被至少一条「投诉」所关联的订单的**实付金额**（去重后合并） |
| **分母** | 现金流收入 | 同一统计周期内，公司第一曲线**统一现金流收入**（BI 口径） |
| **投诉** | 最小计数单位 | 单条**投诉记录**：满足「渠道 + 内容 + 意图」三条件的**一条**用户反馈 |

**原则**：先定义「什么算一条投诉」→ 再定义「投诉如何关联到订单」→ 再定义「订单金额如何汇总进分子」。口径一经发文锁定，未经 BSC/经营会批准不得调整。

---

## 二、投诉的最小颗粒度定义（一条投诉 = 1 条记录）

### 2.1 总体定义（一句话）

**投诉**：用户通过指定渠道发出的、在内容上**同时满足「表达不满」与「表达诉求」**、且**诉求程度为 L1 或 L2**（已转化为可留痕行为，见 2.2.1）的**单次反馈**，以**工单/记录创建时间**为准，**一条记录计为一条投诉**，不得事后删除或改分类。仅情绪化提及、未转化行为（L0）**不计入**百万元投诉率分子。

### 2.2 三条件（必须同时满足）

| 条件 | 说明 | 最小颗粒度规则 |
|------|------|----------------|
| **① 渠道** | 用户反馈所进入的**系统或平台** | 见下《内部渠道清单》《外部渠道清单》；未进入清单的不计入。 |
| **② 内容·表达不满** | 对喜播相关的人/事/物表达不满 | 涉及以下任一类即视为「表达不满」：<br>· 宣传/广告/素材（夸大、虚假、误导）<br>· 课程/内容/师资（质量、与承诺不符）<br>· 服务/流程/人员（态度、时效、未兑现）<br>· 软硬件/产品（无法使用、与描述不符） |
| **③ 内容·表达诉求** | 明确表达「要结果」的意图 | 满足以下**至少一项**即视为「表达诉求」：<br>· **提及退款**（含「退钱」「退费」「退课」「取消订单」等）<br>· **提及投诉/维权**（含「投诉」「举报」「维权」「找监管/媒体」等）<br>· **威胁外部升级**（含「去黑猫/315/消协/市监」等） |

**不纳入**：仅抱怨、仅建议、仅咨询（未同时表达诉求）的反馈，**不计入投诉**；若同一用户在同渠道对同一事由多次留言，按**单工单/单会话**计 1 条，不按句数重复计。

### 2.2.1 投诉程度颗粒度（意图强度：区分「气话」与「真实诉求」）

用户说「要退费」「去投诉」有时是**情绪宣泄**（类似吵架时说「离婚」不一定是真要离），有时是**真实意图**。为减少误计、避免指标虚高，在「表达诉求」基础上增加**程度**维度：仅当诉求**已转化为可留痕的行为**时，才计入百万元投诉率分子；仅口头提及、未转化行为的，**不计入分子**（可单列预警，不进入公式）。

| 程度等级 | 定义 | 是否计入百万元投诉率分子 | 可落地判定规则（最小颗粒度） |
|----------|------|--------------------------|------------------------------|
| **L0 仅情绪化提及** | 用户在沟通中说了退费/投诉等词，但**未在约定时间内转化为任何可留痕的后续动作** | **不计入** | 单次会话/单次接触中仅有文字或口头「要退费」「要投诉」等表述，且在该次会话结束后 **72 小时内**：未发起退费申请、未创建投诉类工单、未到外部平台提交。以**系统是否有退费申请记录/工单创建/外投登记**为准。 |
| **L1 已转化为内部行动** | 用户已发起退费申请、或已创建明确投诉/维权类工单、或已在内部渠道**正式提交**退费/投诉请求（可留痕） | **计入内部投诉** | 存在以下任一条即计：已点击/提交「退费申请」或「退款」按钮并生成记录；已创建类型为「投诉/维权/退费」的工单；NPS/满意度中勾选「要求退费/投诉」或开放题中明确写「申请退款」并提交。 |
| **L2 已转化为外部行动** | 用户已在 12315/黑猫/消费保等外部渠道**提交**投诉或维权案件 | **计入外部投诉** | 以外部平台**案件/投诉单已创建**为准（不论用户是否事后撤销）。 |

**操作细则**：

1. **内部渠道（企微/客服会话等）**  
   - 当次会话中用户仅说「我要退费」「不然投诉」等，**未在 72 小时内**在系统内发起退费申请或创建投诉/退费类工单 → 视为 **L0**，**不计入**。  
   - 当次会话中用户**明确要求「申请退款」「给我办退费」**且客服/系统已按「退费/投诉」流程建单或登记 → 视为 **L1**，**计入**。  
   - 若用户当次会话未建单，但在 **72 小时内**同一用户在同一业务/订单上发起了退费申请或创建了投诉类工单 → 将该会话**补计为 L1**（以工单/申请创建时间为归属周期）。

2. **外部渠道**  
   - 用户已在黑猫/12315/消费保等**提交**案件或投诉单 → 一律视为 **L2**，**计入**（不区分「气话」与否，因已形成公开与监管留痕）。

3. **边界**  
   - 「明确要求申请退款/给我办退费」：用户用肯定、请求类表述（如「我要申请退款」「请给我办退费」「必须退」），而非仅抱怨（如「真是想退钱了」「说真的想投诉了」且未接任何申请动作）→ 前者计 L1，后者若 72 小时内无后续动作计 L0。  
   - 争议处理：单条记录是否属 L0 由客户体验部按上述规则做抽检与复核；系统优先按「是否有退费申请/投诉工单/外投登记」自动判断，减少人为裁量。

### 2.3 内部渠道清单（计入「内部投诉」，40% 权重）

| 序号 | 渠道名称 | 最小颗粒度单位 | 数据来源/系统 |
|------|----------|----------------|----------------|
| 1 | 企微/私域客服 | 单次会话中首次出现「不满+诉求」的**单条会话**（同一会话内不重复计） | 企微/客服系统，按会话 ID 去重 |
| 2 | 喜播学堂/小程序客服 | 单次会话中首次出现「不满+诉求」的**单条会话** | 客服系统，按会话 ID 去重 |
| 3 | 喜马拉雅主站客服（教育相关入口） | 单条**工单**（同一工单不重复计） | 主站客服工单系统，按工单 ID 去重 |
| 4 | NPS/满意度调研 | 单份**问卷**中，在开放题或选项里同时表达「不满+诉求」的**单份答卷** | 调研系统，按答卷 ID 去重 |
| 5 | 其他公司认可的内部反馈入口 | 以**单条工单/单次会话/单份答卷**为单位 | 由客户体验部列清单并纳入 BI 取数 |

**归属时间**：以**工单创建时间 / 会话开始时间 / 答卷提交时间**所在自然月/自然年，计入该周期。

### 2.4 外部渠道清单（计入「外部投诉」，60% 权重）

| 序号 | 渠道名称 | 最小颗粒度单位 | 数据来源/说明 |
|------|----------|----------------|----------------|
| 1 | 全国 12315 / 地方市监投诉 | 单件**投诉案件**（一案一号） | 市监/12315 系统或对接数据，按案件编号去重 |
| 2 | 黑猫投诉 | 单条**投诉单** | 黑猫平台抓取或人工录入，按投诉单 ID 去重 |
| 3 | 消费保 | 单条**投诉单** | 消费保平台抓取或人工录入，按投诉单 ID 去重 |
| 4 | 其他监管/消协/公开平台（经公司发文认定） | 单条**投诉/案件** | 由客户体验部列清单，BI 统一取数 |

**归属时间**：以**投诉单/案件登记时间**（或平台显示时间）所在自然月/自然年，计入该周期。

### 2.5 边界与排除（防口径被放大或缩小）

| 情形 | 是否计入 | 说明 |
|------|----------|------|
| 同一用户同一事由，先走内部再走外部 | **内部计 1 条，外部计 1 条** | 内外双计，体现「防外溢」价值；若内部已关闭且用户明确满意后仍到外部重复投诉，外部仍计 1 条。 |
| 同一用户同一事由，多渠道重复提交 | **按渠道分别计**，但同一渠道内按 2.3/2.4 最小单位去重 | 例如同一人在黑猫投 1 条、消费保投 1 条，计 2 条外部投诉。 |
| 非喜播业务（如喜马拉雅主站非教育） | **不计入** | 仅第一曲线训练营/硬件/喜播品牌相关。 |
| 纯竞品/第三方责任，且经核实与喜播无涉 | **不计入** | 需留档说明，并纳入季度口径评审。 |
| 用户撤销/和解后删除的工单 | **仍计入** | 以**创建时**状态为准，不得因事后撤销或删除而从统计中移除。 |

## 三、投诉 → 订单金额的关联规则（分子如何算）

### 3.1 关联对象

- **订单**：第一曲线训练营、硬件等**实付订单**（以 BI 订单主表为准）。
- **关联规则**：一条投诉只关联**该用户**在**该投诉所涉业务/课程/商品**上的**未退款或部分退款后剩余**的订单金额；若投诉未指向具体订单，则关联该用户**在投诉创建日之前 180 天内**的、同一业务线内**全部实付订单**（同上，扣除已退款部分）。

### 3.2 最小颗粒度规则（订单维度）

| 规则项 | 说明 |
|--------|------|
| **用户唯一标识** | 以手机号/用户 ID 等 BI 统一 ID 为准，同一用户的多条投诉分别关联其订单，**同一订单可被多条投诉关联，但分子中该订单金额只计一次**（按订单去重）。 |
| **金额口径** | 订单**实付金额**（不含优惠券抵扣等已不计入收入的部分）；若已部分退款，则用**实付 − 已退金额**；若已全额退款，该订单**不**计入分子。 |
| **时间窗口** | 投诉关联的订单，以**投诉创建日（或登记日）向前 180 天内**的订单为限；超过 180 天的订单不因该条投诉计入分子。 |
| **合并计算** | 同一统计周期内，先汇总所有「被至少一条投诉关联上的订单」及其**去重后**的订单金额，再按**内部投诉关联订单金额**与**外部投诉关联订单金额**分别汇总，得到「内部投诉订单金额」「外部投诉订单金额」。 |

### 3.3 分子汇总公式（与公式对齐）

- **内部投诉订单金额** = Σ（在统计周期内创建的、渠道属「内部」的投诉所关联的订单，按订单去重后的实付净额）。
- **外部投诉订单金额** = Σ（在统计周期内登记的、渠道属「外部」的投诉所关联的订单，按订单去重后的实付净额）。
- 若同一订单既被内部投诉关联又被外部投诉关联，**该订单金额同时计入内部与外部分子**（与 2.5 中「内外双计」一致）。

## 四、分母：现金流收入

- **口径**：BI 提供的**公司第一曲线统一现金流收入**（训练营+硬件等，不含内部调账），与净营收/经营分析口径一致。
- **周期**：与分子统计周期一致（月度/季度/年度）。
- **约束**：分母由 BI 统一输出，业务部门不得自行调整。

## 五、统计周期与发布

| 项目 | 约定 |
|------|------|
| **周期** | 月度、季度、年度均可算；**百万元投诉率 BSC 目标**按**年度**及**同比/环比**使用，内部管理可按月看趋势。 |
| **发布** | 由 BI 按本口径输出「内部投诉订单金额」「外部投诉订单金额」「现金流收入」及**百万元投诉率**（含内部分项、外部分项），客户体验部负责口径解释与异常复核。 |
| **审计** | 每季度抽检：投诉记录是否满足「渠道+内容+意图」三条件；订单关联是否满足 3.1～3.3；不得事后删除或改分类以改变统计结果。 |

---

## 六、口径变更与例外

- **变更**：任何对「投诉」定义、渠道清单、关联规则、分母的调整，须经**BSC/经营会**批准并**发文**后生效，且需注明生效周期，避免追溯篡改历史数据。
- **例外**：个别案件确属误记（如误点、测试数据），由客户体验部提报，**仅可做「标注不参与考核」**，不得从底层记录删除或改分类；BI 可提供「含/不含标注例外」两套结果供管理参考。

---

## 七、与战略目标的对齐

| 战略目标 | 口径设计如何支撑 |
|----------|------------------|
| 降低「带风险的收入」占比 | 分子用**投诉关联订单金额**，直接度量「可能因投诉/退费而流失」的收入。 |
| 内外双口径防变形 | 60% 外部 + 40% 内部，避免只压内部不解决外投；外部权重更高，倒逼防外溢。 |
| 可执行、可审计 | 最小颗粒度 = 单条记录、单渠道去重、订单关联规则明确，BI 可落库、审计可抽检。 |
| 防少记、防改分类 | 以创建时状态为准、不得事后删除或改分类、例外仅标注不删不改。 |
